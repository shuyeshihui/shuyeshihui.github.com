需求2：编辑器前端的字数统计规则确定

目前的规则：汉字、英文字母、数字、标点符号字符、空格都是1：1换算成字数的；最大字数限制为15000

改进的规则：最大字数限制从15000改为最大字数限制为25000

需求3：复制粘贴到编辑器中的内容格式如果不支持，去除格式只保留内容文字

目前的规则：


1、目前编辑器里统计字数和复制过来的文章本身所带的格式，在前端统计确实只统计字数，但是后端统计做了字数＋格式的控制，控制的字数为5万字

2、目前实际文章在APP中支持的格式为：


正文样式支持：1. 加粗 2. 斜体 3.下划线 4. 字体颜色 5.三级标题样式 6.有序列表 7.无序列表 8.图片


正文支持的操作：空格，手动换行，自动换行

改进的规则：


前端拷贝的文章去掉本身内容不支持的格式（1. 加粗 2. 斜体 3.下划线 4. 字体颜色 5.三级标题样式 6.有序列表 7.无序列表 8.图片


和操作：空格，手动换行，自动换行），不支持的格式只保留文本内容


PS：

1、表格的样式目前也不支持，所以复制表格样式的粘贴到编辑器中呈现的是表格中的文字；

2、自动换行和手动换行的行间距应该是不一样的，目前是一样的
