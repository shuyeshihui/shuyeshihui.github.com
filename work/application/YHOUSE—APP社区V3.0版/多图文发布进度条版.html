网络不给力，发布失败

是否确定要删除？

是

否

点击重发则立即重发

当用户点击了发布则立马跳转到关注界面，在关注界面上方显示发送进度条

点击取消则会弹出框来让用户确认是否要取消发送。

 

还没有发送出去，出现网络异常时，重发机制都试过了，则会显示发送失败。“取消”按钮会变成“重发”按钮，点击重发则断点续传。

如果用户在发送过程中继续发其他的图文会在该进度条的下方继续出现下一个图文的进度条，队列还是先发第一条再发第二条，第一条发送失败了才会尝试第二条的发送。以此类推。

取消

取消

消息的重发机制：消息的重发设为每条消息发X次，X次失败后则会发送失败；假设此时有两条在发，第一条重发机制试完了失败则第二条也默认为失败。

用户在发送图文的界面点击发送，如果没有网络连接就不会跳转过来；直接在发送图文的界面显示提醒，没有网络连接

用户在关注界面等待图文的发送时，用户切换到其它应用，后台也会默默的执行发送。如果发送成功即发送成功，发送失败则按进度条界面显示网络不给力，发布失败。后续会将发送失败和消息系统进行打通，告诉用户有图文发送失败。

用户在关注界面等待图文的发送时，用户把该应用的进程给结束，目前可以先不做草稿箱的功能，默认该消息丢弃。之后可以放到草稿箱里。
